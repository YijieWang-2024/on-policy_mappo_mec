# 2026-07-09 Fixed-K Final Heldout Evaluation

## Purpose

Produce the main same-scenario comparison at the paper's fixed fleet size K=16.
This evaluation increases environment coverage and stores per-episode metrics
for paired and hierarchical-bootstrap analysis.

## Protocol

- Scenario: `v7_random_split_hotspots`
- Fleet size: K=16
- Methods: `flat`, `mean`, `hotspot_pool`, `anchor_pool`, `csd_pool`
- Training seeds: `{1, 2, 3}`
- Checkpoint selection: best validation checkpoint from each 3.5M run
- Final heldout episodes: 100 per checkpoint
- Final heldout seeds: `200000 + 13i`, disjoint from validation and prior heldout
- Concurrent evaluations: at most 3
- Output includes aggregate means and all episode-level rows

Run:

```powershell
powershell -ExecutionPolicy Bypass -File experiments\2026-07-09_fixed_k100_eval\run.ps1
```

Outputs:

`artifacts/evaluations/v7_random_split_hotspots/fixed_k100_20260709/`
